## 🏨 About The Project

A **command-line Hotel Reservation System** built entirely in **Core Java** as part of an internship project. The system enables users to search, book, and manage hotel room reservations categorized as **Standard, Deluxe, or Suite** — complete with simulated payment processing and persistent file-based data storage.

> 🎓 Built to demonstrate solid understanding of **Object-Oriented Programming**, **file I/O**, and **modular Java architecture** — no external libraries required.

<br/>

<div align="center">

| 🏨 Rooms | 👤 Guests | 💳 Payment | 📁 Storage |
|:---:|:---:|:---:|:---:|
| Standard · Deluxe · Suite | Name & phone booking | Simulated processing | `rooms.txt` & `bookings.txt` |

</div>

## ✅ Features

<br/>

| 🏨 Reservation Management | 💳 Payment & History | 📁 Data Persistence |
|:---|:---|:---|
| Room categorization — Standard, Deluxe, Suite | Simulated payment processing | All data stored in `rooms.txt` |
| Search available rooms by type | View full booking history | Guest records in `bookings.txt` |
| Book a room with guest name & phone | Cancel reservation by room number | File I/O — no database required |

## 📁 Project Structure

```
Hotel-Reservation-System-Java/
│
├── 📂 src/main/
│   ├── 📄 HotelSystem.java           ← Entry point & CLI menu logic
│   ├── 📂 model/
│   │   ├── Room.java                 ← Room entity (type, number, availability)
│   │   ├── Booking.java              ← Booking record model
│   │   └── Guest.java                ← Guest details model
│   └── 📂 util/
│       ├── DataManager.java          ← File I/O read/write operations
│       └── PaymentSimulator.java     ← Simulated payment processing
│
└── 📂 data/
    ├── rooms.txt                     ← Room data (auto-loaded on start)
    └── bookings.txt                  ← Booking records (can start empty)
```

## 🚀 How to Compile & Run

**Compile**

```bash
javac -d bin -sourcepath src src/main/HotelSystem.java
```

**Run**

```bash
java -cp bin main.HotelSystem
```

> ✅ Ensure the `data/` folder exists with `rooms.txt` populated. `bookings.txt` can start empty.
